# Render Function

The render layer opens links generated by `/create` and decodes payloads.

## Main routes

- `/r` and `/ra` for compact generic rendering
- `/render` and `/render-all` as full route variants
- `/render/image`, `/render/pdf`, `/render/video`, `/render/audio`, `/render/office` for specialized viewers

## How rendering works

1. Read payload from hash/query (`#d=` or `?d=`)
2. Detect content type marker
3. Decompress and decode content
4. Send content to the correct viewer
5. Show output in standard mode or fullscreen mode

## Compatibility details

- Keeps support for legacy markers (`#data=` and `?data=`)
- Supports shared-link mode with `?z=1` and `?z=0`
- Works with static hosting deep links via `404` recovery
